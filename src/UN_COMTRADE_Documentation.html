Query Modes

Trade Summary

Aggregate trade totals for one reporter against the World. Shows total import and export values per period.

Column Description
Period Year (or month if Monthly frequency)
Reporter / Reporter ISO Reporting country name and ISO 3-letter code
Flow Import, Export, Re-import, or Re-export
Trade Value (USD) Total trade value in US dollars
Net Weight (kg) Total weight; may be 0 for aggregate TOTAL commodity level (only populated for specific commodities)
Unit Value (USD/kg) Trade Value ÷ Net Weight. Empty when Net Weight is 0 or unavailable.
YoY Growth (%) Year-over-year percentage change vs. the previous year. Only appears when multiple years are selected.

Trade Balance

Exports and imports side by side for a reporter–partner pair, with a computed balance column. Uses the dedicated UN Comtrade balance endpoint.

Column Description
Exports (USD) Total export value
Imports (USD) Total import value
Trade Balance (USD) Exports − Imports. Positive = surplus, negative = deficit.

Partner Breakdown

Trade values with every partner country. Useful for identifying top trading partners. The “World” row shows the aggregate total.

Column Description
Partner / Partner ISO Partner country name and ISO code
Trade Value (USD) Bilateral trade value with this partner
Trade Share (%) This partner's share of total trade (excluding the World aggregate row) within the same period and flow direction.

Product Breakdown

Trade values by commodity code for a specific reporter–partner pair. Supports 2-digit chapters, 4-digit headings, or specific codes.

Column Description
Commodity Code / Commodity HS, SITC, or BEC code and description
Trade Share (%) This commodity's share of total trade within the same period and flow.
Unit Value (USD/kg) Price per kilogram; useful for comparing price trends.

Bilateral Trade

Detailed commodity-level trade between two specific countries. Optionally uses mirror data (the partner's reported data) as a fallback when the reporter has no data.

Column Description
Data Source reported = reporter's own data; mirror = partner's data with flows flipped (only when “Use Mirror Data” is enabled).
YoY Growth (%) Year-over-year change per commodity–flow combination across periods.

General Notes

Net Weight is only available for individual commodities. At the aggregate TOTAL level, Net Weight will be 0 and Unit Value will be empty. This is expected.
Trade Share is calculated within each period + flow group. In Partner Breakdown, the “World” row is excluded from the denominator and shows no share value, since it represents the total itself.
YoY Growth matches rows across years using stable identifiers (ISO codes, commodity codes, flow). It requires at least two periods. The first year in the range will not have a growth value.

Classification Systems

HS (Harmonized System) — the most common goods classification. Revised every 5 years (HS92, HS96, HS02, HS07, HS12, HS17, HS22).

SITC (Standard International Trade Classification) — analytical classification grouping products by economic function.

BEC (Broad Economic Categories) — groups products into capital goods, intermediate goods, and consumer goods.

EBOPS (Extended Balance of Payments Services) — used for Services data type.

Tips

• Start with Trade Summary to get the big picture, then drill down with Partner or Product Breakdown.

• For bilateral trade with small or developing countries, enable Use Mirror Data — the partner may report data even when the reporter does not.

• Use Annual frequency for most analyses. Monthly data produces many more rows and is only available for Goods.

• Each fetch creates a new sheet. Use UN Comtrade > Clear All Cache if metadata seems stale.
