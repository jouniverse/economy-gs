About UN Comtrade

The UN Comtrade Database is the world’s largest repository of international trade data, maintained by the United Nations Statistics Division. It covers merchandise (goods) trade from 1962 and services trade from over 200 countries and territories.

This add-on queries the UN Comtrade API for trade data. Select a country, period, and analysis mode, then inject the results into a Google Sheet.

Query Modes

Mode Description
Trade Summary Total trade value for one reporter against the World. Shows aggregate import/export values per period.
Trade Balance Exports and imports side by side with a calculated balance column (Exports − Imports).
Partners Trade values by partner country with trade share percentages. The “World” row shows the total.
Products Trade values by commodity code with unit values and shares. Use 2-digit for overview, 4-digit for detail.
Bilateral Commodity-level trade between two specific countries. Supports mirror data fallback.

Output Columns

Column Notes
Trade Value (USD) Value in US dollars as reported to UN
Net Weight (kg) Only populated for specific commodities; 0 at TOTAL level
Unit Value (USD/kg) Trade Value ÷ Net Weight; blank when weight is 0
Trade Share (%) Share within same period & flow. World row excluded from denominator in Partners mode.
Trade Balance (USD) Exports − Imports. Positive = surplus.
YoY Growth (%) Year-on-year change; requires ≥2 periods selected
Data Source reported = reporter's data; mirror = partner's data with flows flipped

Classifications

Goods:

  • HS (Harmonized System) — most common; revised every 5 years (HS92 to HS22)
  • SITC — analytical classification by economic function (since the 1950s)
  • BEC (Broad Economic Categories) — groups products by end-use (capital, intermediate, consumer)

Services:

  • EBOPS 2010 — current services classification
  • EBOPS 2002 — older version, still used by some countries
  • EBOPS (Combined) — merged; coverage varies
Services data coverage is significantly sparser than goods. Many countries do not report services trade. You may need to try different classifications and commodity levels to find available data.

Tips

  • Start with Trade Summary to get the big picture, then drill down with Partners or Products.
  • Use Annual frequency for most analyses; monthly is goods only and produces many more rows.
  • For bilateral trade with smaller countries, enable Mirror Data — the partner may report data even if the reporter does not.
  • Set Partner to World for aggregate views. Avoid “All Partners + 4-digit products” — this can hit API record limits.
  • Each fetch creates a new sheet. Use the API key stored in Script Properties to avoid re-entering it.
  • The free API tier allows 100k records per call. Use aggregate commodity codes to stay within limits.
